# SellerOffice (SO-FE) — NFT 운영 관리자 플랫폼

## 프로젝트 개요

NFT 및 디지털 자산 운영을 위한 관리자(Admin) 플랫폼 프로젝트입니다.

운영자가 매장(Shop) 등록, NFT 등록 및 관리, 컬렉션 생성, 대량 업로드, 파일 관리 등을 효율적으로 수행할 수 있도록 구축한 Web 기반 Admin 서비스입니다.

Vue 3 기반으로 개발되었으며, NFT 등록 과정에서 이미지/비디오 파일 처리, 대량 업로드, 메타데이터 관리 등 복잡한 기능을 구현한 것이 특징입니다.

## 주요 기능

### 1. NFT 등록 및 관리 기능

NFT 데이터를 등록하고 관리할 수 있는 기능을 구현했습니다.

### 제공 기능

- NFT 생성
- NFT 정보 수정
- 컬렉션 생성
- 메타데이터 입력
- 이미지/비디오 업로드

### 구현 포인트

- 다단계 Form 구조 설계
- Validation 기반 입력 처리
- 부모-자식 컴포넌트 상태 관리
- 동적 Form 데이터 처리

---

### 2. 대량 업로드(Bulk Upload) 기능

여러 개의 NFT를 한 번에 등록할 수 있는 대량 업로드 기능을 구현했습니다.

### 구현 기능

- 다중 파일 업로드
- 업로드 상태 관리
- 업로드 진행 단계 표시
- 업로드 실패 처리
- 메타데이터 일괄 처리

### 구현 포인트

- Step 기반 업로드 구조 설계
- 대량 데이터 상태 관리
- 비동기 업로드 흐름 처리
- 업로드 Validation 처리

---

### 3. 파일 업로드 및 미디어 처리

NFT 등록 과정에서 이미지 및 비디오 파일을 처리할 수 있는 업로드 시스템을 구현했습니다.

### 구현 기능

- Drag & Drop 업로드
- 이미지/비디오 미리보기
- 파일 유효성 검증
- 파일 삭제 및 상태 관리
- 동영상 처리 및 변환

### 구현 포인트

- File API 기반 업로드 처리
- URL.createObjectURL 활용
- Object URL 메모리 관리
- 재사용 가능한 Upload 컴포넌트 설계

---

### 4. FFmpeg 기반 비디오 처리

NFT 등록 시 업로드되는 비디오 파일을 FFmpeg 기반으로 처리하는 기능을 구현했습니다.

### 구현 기능

- 비디오 포맷 처리
- 썸네일 생성
- 미디어 변환 처리

### 구현 포인트

- FFmpeg 연동
- 비동기 처리 흐름 설계
- 업로드 상태와 연계된 처리 구조 구현

## 트러블 슈팅

### 부모-자식 컴포넌트 상태 관리 문제

### 문제

복잡한 Form 구조에서 컴포넌트 간 상태 동기화가 어려운 문제가 발생

### 해결

- v-model 기반 데이터 흐름 통일
- defineExpose 활용
- 공통 Form 구조 설계

---

### 대량 파일 업로드 메모리 문제

### 문제

다량의 이미지 및 비디오 Preview 처리 과정에서 메모리 사용량 증가

### 해결

- Object URL revoke 처리
- Preview 렌더링 최적화
- 상태 관리 구조 개선
